# 📘 Technical Invariants Catalog — RentGuard AI

Esta especificación detalla las restricciones técnicas y políticas de exactitud de datos que deben cumplirse en todas las capas del sistema.

---

## 1. Precisión Financiera (Rounding Policy)

| Dato | Precisión | Política de Redondeo |
| :--- | :--- | :--- |
| **Monto de Pago** | 2 Decimales | Round Half Up (Hacia el más cercano) |
| **Balance Mensual** | 2 Decimales | Round Half Up |
| **Tasa / Índice** | 4 Decimales | Truncamiento |

---

## 2. Integridad de Identidad y Seguridad

- **Regla TI-01 (Hash Normalization)**: El SHA-256 de los comprobantes debe generarse sobre el archivo binario original sin metadatos EXIF para evitar colisiones por rotación de imagen.
- **Regla TI-02 (WhatsApp Uniqueness)**: El número de teléfono se normaliza al formato E.164 antes de cualquier comparación o búsqueda.

---

## 3. Límites Transaccionales

- **TI-03 (Isolation Scope)**: Toda transacción SQL debe operar bajo el nivel de aislamiento `READ COMMITTED SNAPSHOT` (por defecto en SQL Azure/SQL Server moderno) para evitar bloqueos de lectura.
- **TI-04 (Atomic Batch)**: El procesamiento de un batch de Outbox no debe exceder los 100 mensajes por ciclo para evitar agotamiento de memoria del Worker.

---

## 4. Invariantes de Esquema

- **TI-05 (Immutable IDs)**: Los Primary Keys (GUID) son inmutables tras la creación.
- **TI-06 (Auditable)**: Toda tabla de dominio **DEBE** poseer las columnas: `CreatedUtc`, `LastUpdatedUtc`, `CreatedBy` y `RowVersion`.

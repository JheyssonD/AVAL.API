## 1. Visión Arquitectónica (The Blueprint)

RentGuard AI se construye sobre una arquitectura de **Sistemas Distribuidos Resilientes** utilizando patrones de diseño de nivel Enterprise para garantizar la integridad financiera y el aislamiento multi-tenant absoluto.

### 1.1 Patrones Core
- **CQRS**: Separación estricta de comandos (escritura) y consultas (proyecciones de lectura).
- **Domain-Driven Design (DDD)**: Modelado basado en agregados con límites de contexto claros.
- **Multi-Tenancy (Strategy Pattern)**: Aislamiento físico y lógico de datos mediante `TenantId` y Global Query Filters.
- **Event-Driven Architecture (EDA)**: Comunicación asíncrona garantizada mediante el patrón **SQL Outbox** (Consistencia Eventual).

## 3. Pilares de Integridad Técnica

### 3.1 TrustScore (Sistema Reputacional)
- **Cold Start**: 500 pts (Neutral).
- **Rango**: [0, 1000].
- **Lógica**: Basada en vectores de tendencia (Regresión Lineal) para diferenciar trayectorias.

### 3.2 Gestión de Pagos (FSM)
- **Terminalidad**: `Approved`, `Rejected`, `Duplicate` y `PermanentFailure` son estados finales.
- **Idempotencia**: Short-circuit success (200 OK) para webhooks de Meta ante mensajes duplicados.

### 3.3 Aislamiento y Seguridad (Zero Trust)
- **Backend**: Acceso a Blobs mediante Managed Identity (RBAC) para resiliencia en reintentos.
- **Frontend**: Acceso vía SAS Tokens de 5 minutos.
- **Base de Datos**: Filtrado global forzado por `TenantId`.

---

## 4. Estándares de Ejecución (DoD)

1. **Clean Code**: Inyección de dependencias, interfaces para infraestructura y tipos inmutables.
2. **Resiliencia**: Retries exponenciales nativos en SQL Server (`UPDLOCK, READPAST`).
3. **Comunicación**: Errores bajo estándar **RFC 7807** y eventos asíncronos atómicos.
4. **Calidad**: Cobertura de integración obligatoria para flujos de `ITenantContext`.

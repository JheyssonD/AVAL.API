# 📘 Event Contracts Specification — RentGuard AI

Esta especificación define el estándar de comunicación asíncrona. Todos los eventos de dominio publicados en el Outbox deben cumplir con este contrato para asegurar la interoperabilidad y el desacoplamiento.

---

## 1. Esquema Base (Envelope)

```json
{
  "eventId": "guid",
  "eventType": "string",
  "version": "integer",
  "tenantId": "guid",
  "correlationId": "guid",
  "causationId": "guid",
  "occurredOnUtc": "datetime",
  "payload": {
    "//": "Campos específicos del evento"
  }
}
```

---

## 2. Catálogo de Eventos v1

### 2.1 PaymentApproved
Disparado cuando un pago es validado y aplicado al balance.
- **Payload**: `{ "paymentId": "guid", "leaseId": "guid", "residentId": "guid", "amount": 1000.00, "currency": "USD" }`

### 2.2 PaymentRejected
Disparado cuando un comprobante es invalidado.
- **Payload**: `{ "paymentId": "guid", "reason": "string", "isFraud": "boolean" }`

### 2.3 TrustScoreUpdated
Disparado tras el recálculo de reputación.
- **Payload**: `{ "residentId": "guid", "previousScore": 500, "newScore": 515, "reason": "string" }`

---

## 3. Políticas de Gobernanza

### 3.1 Versionado Semántico
- **Cambiables**: Adición de campos (Compatible).
- **Incompatibles**: Renombrado o eliminación de campos (Incremento de `version`).
- **Policy**: El sistema debe soportar al menos la versión `N` y `N-1`.

### 3.2 Convención de Nomenclatura
- **Formato**: `Entity` + `PastAction` + `Event` (ej. `LeaseActivatedEvent`).
- **Case**: PascalCase para el tipo de evento, camelCase para los campos del JSON.

### 3.3 Serialización
- **Formato**: JSON (UTF-8).
- **ISO-8601**: Formato obligatorio para todas las fechas en UTC.

# 📘 Error Catalog Specification — RentGuard AI

Esta especificación cataloga todos los errores posibles del sistema, definiendo su semántica, severidad y el comportamiento esperado por los consumidores (API, SPA, WhatsApp).

---

## Context Boundary
Este documento es la **fuente de verdad de fallos**. Define la taxonomía y códigos de error únicos. No define la lógica de recuperación de workflows (ver 007) ni las políticas de seguridad (ver 004), sino la comunicación estandarizada de estados de error hacia los clientes.

---

## 1. Errores de Dominio (Business Errors)

| Código | Mensaje | Retryable | HTTP | Descripción |
| :--- | :--- | :--- | :--- | :--- |
| **DOM-409-DUP** | Duplicate Voucher/Message | No | 409* | Hash ya procesado. *Nota: En Webhooks Meta, responder 200 OK (Silent Success) para evitar DoS por reintento. |
| **DOM-400-EXP** | Lease Already Expired | No | 400 | Intento de pago en contrato cerrado. |
| **DOM-422-INS** | Insufficient Payment Amount | No | 422 | Monto menor al mínimo requerido. |
| **DOM-403-ISO** | Multi-Tenancy Violation | No | 403 | Acceso a datos de otro TenantId. |

---

## 2. Errores de Infraestructura (Technical Errors)

| Código | Mensaje | Retryable | HTTP | Descripción |
| :--- | :--- | :--- | :--- | :--- |
| **INF-503-OCR** | OCR Provider Timeout | **Sí** | 503 | Fallo temporal en Vision AI. |
| **INF-500-DB** | Database Connection Failed | **Sí** | 500 | SQL Server no disponible. |
| **INF-413-SIZE** | Voucher Too Large | No | 413 | Imagen excede los 10MB. |
| **INF-400-COR** | Corrupt Payload | No | 400 | El archivo recibido está dañado. |

| Código | Mensaje | Retryable | HTTP | Descripción |
| :--- | :--- | :--- | :--- | :--- |

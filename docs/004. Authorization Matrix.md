# 📘 Authorization Matrix (RBAC) — RentGuard AI

Esta matriz define los permisos de acceso para cada rol dentro del ecosistema RentGuard AI. La seguridad se basa en la validación de claims JWT y el aislamiento de datos por `TenantId`.

---

## 1. Matriz de Acciones

| Acción | Resident | Landlord | Global Admin |
| :--- | :--- | :--- | :--- |
| **Report Voucher (WhatsApp)** | ✅ Own | ❌ | ❌ |

---

## 2. Reglas de Aislamiento (Mandatory Rules)

1. **Own Scope**: El Landlord solo puede ejecutar acciones sobre entidades que contengan su `TenantId`. Toda solicitud sin `TenantId` o con uno ajeno será rechazada con un error `SEC-403-PERM`.
2. **Cross-Tenant Access**: Queda estrictamente prohibido el acceso a datos de otros tenants para los roles Resident y Landlord. Solo el **Global Admin** tiene esta capacidad, la cual debe estar respaldada por un ticket de soporte/auditoría.
3. **Identity Propagation**: El `ResidentId` y el `TenantId` deben estar codificados en el JWT del usuario tras la autenticación y no deben ser modificados por el cliente.
